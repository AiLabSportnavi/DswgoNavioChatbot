What we built

One assistant. Three ways to use it.

  • Floating widget — a chat bubble mounted directly on the SportNavi landing page and hub.
  • Embeddable script — a single <script> line drops Navio into any external site, fully style-isolated.
  • Admin console — the team edits Navio's behaviour (system prompt) live, no redeploy needed.

Powered by Azure OpenAI (GPT-4.1) and verified end-to-end in production.

How it works

A clean, stateless architecture

Frontend

React 19 + Vite

Tailwind v4, Motion. Widget + admin hub. Served via Nginx.

→
Backend

FastAPI

/api/chat to Azure GPT-4.1. Rate-limited, validated, stateless.

→
Data

Supabase

Live config + consent-gated, anonymised conversation logs.

→
Hosting

Google Cloud Run

Frankfurt (EU). Auto-scaling containers, secrets managed.

Stateless by design — the browser stays same-origin, the backend URL stays hidden, and there's no CORS in the user's path.

Trust & Compliance

Privacy isn't a feature.
It's the foundation.

  • Explicit consent before any message is processed — and again after every reset.
  • Metrics-only by default — message content is stored only when the visitor consents.
  • No raw IPs — addresses are one-way hashed; sessions are anonymised tokens.
  • EU data residency — hosted in Frankfurt, links straight to sportnavi.de/datenschutz.
