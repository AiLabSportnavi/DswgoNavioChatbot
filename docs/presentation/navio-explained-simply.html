The big picture

What is Navio?

Navio is a friendly chat assistant that lives on the SportNavi website. Visitors type a question, and Navio answers — like a helpful guide who is always available.

💬

Ask anything

A visitor opens the little chat bubble on the website and types a question in their own words.

🧠

Navio thinks

Behind the scenes, Navio sends the question to a powerful AI, gets an answer, and checks it stays on-topic.

✅

Helpful reply

A clear, friendly answer appears in the chat — usually in about a second.

🛡️

The whole thing is built to be safe and trustworthy: it only works on the official SportNavi website, it can't be flooded by bots, and no private keys are ever exposed to visitors.

How it's built

Two parts: the face and the brain

Like a restaurant, Navio has a friendly front-of-house and a busy kitchen. You only ever see the front — the kitchen does the real work out of sight.

🖥️ The Frontend — what people see

The webpage and the little chat bubble. It's the part that looks nice, takes the visitor's question, and shows the answer.

  • › The chat window and SportNavi design
  • › Runs in the visitor's web browser
  • › Holds no secrets — just the friendly interface

🧠 The Backend — the brain

The hidden engine. It checks the question is allowed, talks to the AI, and keeps everything secure and within limits.

  • ›Talks to the AI and applies the rules
  • ›Keeps all the secret keys safely server-side
  • ›Visitors never see or touch it directly

Where Navio lives

Three ways to host Navio

"Hosting" just means: where does the brain actually run? Think of it like deciding where to keep a shop — rent a managed space, run your own building, or set up a stall at home to practice.

OPTION A
☁️

Cloud hosting

The easy, hands-off choice

A big cloud provider runs Navio for you and keeps it online automatically — like renting a fully-managed, serviced office.

OPTION B
🖧

Your own server

Full control, one box

Everything Navio needs runs together on a single server you rent and control — like owning your own building.

OPTION C
💻

Local instance

For testing & building

Navio runs on a developer's own computer, just for trying things out — like a practice kitchen, not open to the public.

All three run the exact same Navio — only the location and who keeps it running change.

Option A

Cloud hosting — hands-off & reliable

This is how Navio runs today. A trusted cloud provider hosts it in a European data centre and automatically keeps it online, even when lots of people visit at once.

  • ☁️Always online — the provider handles servers, updates, and uptime for you.
  • 📈Grows automatically — handles quiet days and busy spikes without any manual work.
  • 🇪🇺European location — hosted in a Frankfurt data centre, keeping data in Europe.
  • 🔑Secrets stay locked away — passwords and keys are stored in a secure vault, never in the website.
  • 🛠️Least to maintain — almost nothing to manage yourself day-to-day.

Best for: the live public website — when you want it reliable and low-effort.

NAVIO · Option A · Cloud hosting5 of 9
Option B

Your own server — everything in one box

A VPS (a rented server you control) where every piece of Navio runs together. One single front door handles security and HTTPS, and the rest stays safely tucked away inside.

  • 📦All-in-one — frontend, brain, database, and traffic guard start together with one command.
  • 🔐Automatic HTTPS — a single "front door" secures the connection and is the only public entry.
  • 🎛️Full control — you own the server and decide exactly how it runs.
  • 🧱Self-contained — the database lives right alongside it, so there are no outside dependencies to set up.
  • 🛠️You maintain it — more control means you look after updates and the server yourself.

Best for: teams who want everything on infrastructure they fully own and control.

NAVIO · Option B · Your own server6 of 9
Option C

Local instance — a private practice run

Navio running on a developer's own laptop, just for them. Nothing is public — it's the safe sandbox for building new features and testing before anything goes live.

  • 🧪Safe to experiment — try changes without affecting the real website or any visitors.
  • ⚡Quick to start — runs out of the box with one command, no setup or accounts needed.
  • 🔒Completely private — only that computer can reach it; nothing is exposed to the internet.
  • 🔁The same Navio — what works here behaves the same way once it's hosted for real.

Best for: developers building, fixing, and testing features before they go live.
